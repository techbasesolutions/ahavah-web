Anti-patterns to avoid

Don't change what wasn't asked

User has flagged unsolicited UI changes repeatedly. When fixing one element, touch only that element. See feedback_stop_unrequested_changes.md in auto-memory.

Don't hand-roll primitives

Every UI element must be a shadcn/Kibo/app component. Two documented exceptions: LogoMark + sticker SVGs.

Don't mark done without observation

Type-check + build = compiles, not looks right. UI changes need browser observation. Type-check alone is insufficient.

Don't use em-dashes in user copy

Per feedback_no_em_dashes — never — in user-facing strings. Use en-dash, hyphen, or rewrite.

Always set git email

All commits use admin@techbaseltd.com. If not configured globally, pass git -c user.email=admin@techbaseltd.com.

Read Next 16 docs first

Per AGENTS.md — before non-trivial Next 16 API usage, read node_modules/next/dist/docs/.
