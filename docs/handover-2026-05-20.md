## 3. Non-negotiable working rules

These come from the user (CLAUDE.md + standing memory). Violating them wastes the user's time and erodes trust.

1. **Plan before code (INVIOLABLE).** For any non-trivial/new feature, write a plan doc and get explicit user approval BEFORE writing code. Small bug fixes the user has already pointed at can proceed.
2. **Kit primitives only.** Every UI element is a kit component (`components/ui/*` shadcn, `components/kibo-ui/*`, `components/app/*`). No hand-rolled atoms. Extend the primitive (e.g. add a `tone`/`size` to the Button cva) rather than ad-hoc `className`. Layout-only divs (flex/grid) are fine.
3. **Source-of-truth-first.** Before changing a visual surface, find the canonical design (`docs/handoff-desktop/screens/*.md`, Figma handoff). Implement from canonical; if none exists, document the gap and propose before coding.
4. **Verify before claiming done.** `npx tsc --noEmit` clean + `npx eslint <files>` clean. Typecheck/lint = compiles, NOT looks-right. **You cannot visually verify auth-gated screens** (no browser/credentials) — say so explicitly rather than claiming UI success.
5. **No em-dashes (`—`) in user-facing strings.** Use commas, hyphens-with-spaces, or split sentences. Code comments may use them.
6. **Commits:** author email is `admin@techbaseltd.com` (already the git config in both repos). Add trailer `Co-Authored-By: Claude Opus 4.7 <noreply@anthropic.com>`. One topical commit per task. **Stage specific files by name** — both repos have piles of untracked noise (screenshots, design folders, `sw.js`) you must NOT commit. Never `git add -A`/`.`.
7. **Deploy via git push**, not manual file copy. Frontend ships independently of backend.
8. **Don't change UI atoms the user didn't ask about.** Match existing patterns in the file; don't silently "improve" icons/buttons/styles. (Recurring past complaint.)
9. **`/matches` cards must show action buttons** (Message + View profile, or Like-back + View profile) — NOT image-only. This has been reverted/re-broken 5+ times; verify it stays.
10. **Migrations must be idempotent** — the deploy re-runs every migration every time and swallows errors. Guard with `IF NOT EXISTS` / `pg_constraint` existence checks / `DROP ... IF EXISTS` then add. (Migration 0014 once failed silently with `ADD CONSTRAINT ... USING INDEX` on a non-unique index, which rolled back the whole token economy and emptied `/discover` for everyone — see its header comment.)

There is a fuller failure-pattern list in `ahavah-web/PROJECT-STATUS.md` §9 and the user's CLAUDE.md (`ahavah-web/CLAUDE.md` + root `d:/Antigravity/CLAUDE.md`). Read `ahavah-web/CLAUDE.md` for the mandatory reading order.

## 5. Architecture notes you'll need

**Token economy (backend).** Append-only `token_ledger`; balance = `SUM(delta)` per `person_uuid` (no counter column). Spend actions live in `service/tokens/actions/<name>.py`, each exporting `perform(tx, person_uuid, …)` that calls `service.tokens.debit(...)` then does its side effect, all inside ONE `api_tx()` (caller owns the transaction). `debit` raises `InsufficientTokens` → endpoints map to 402. Reasons are CHECK-constrained in migration `0014` + `0015` (`purchase, subscription_stipend, reveal_liker, super_like, day_pass, boost, refund, rewind`) — **adding a new reason needs a new idempotent migration** that drops + re-adds `token_ledger_reason_check`. Costs: reveal 1, super-like 2, day-pass 3, boost 5, rewind 1.

**Decisions/swipes (backend).** `POST /decisions {profile_uuid, decision:"like"|"nope"}`. `nope` → `skipped` table (+ upstream `swipe` table, `direction='pass'`). `like` → `liked` table, mutual → `ahavah_match` row + push. Rewind deletes the `skipped` + `'pass'` `swipe` rows (pass-only; never undoes a like). `/discover` is `GET /search` (excludes self, swiped, blocked, messaged; LEFT JOINs `active_boosts`). `swipe_direction` enum = `{like, pass, super}`.

**Chat (frontend).** `chatClient` singleton (`src/lib/chat-client.ts`) = XMPP-over-WebSocket. Lifecycle: open WS → `<open>` → SASL auth → `auth-success` → re-`<open>` → bind → **`ready`** (emit) → live. Connection is warmed app-wide by `useInboxUnreadCount` (mounted in BottomNav + DesktopSidebar). Hooks (`useInbox`, `useChatThread`) fetch on the `ready` event (recently fixed — do not reintroduce delay-after-auth guessing). `isChatTransportAvailable()` gates on mixed-content (`ws://` from `https://` is blocked → shows "Chat ships with domain launch" banner).

**Profiles (frontend).** `useProfile()` → `profile.photos: PhotoRecord[]` (each with `cdn_url` from `cdnUrlFor(uuid)`). `photoOrGradient(profileLike, slot)` → `{kind:"photo",src}` or `{kind:"gradient",css}` (deterministic gradient seeded by lowercased firstName). Use this everywhere a person photo renders; pair with kit `Avatar`/`AvatarFallback`. `ahavah_extra` is an opaque JSONB blob (migration 0009) — backend round-trips it without validating shape, so several profile fields (e.g. multi-select identification/`assembly`) are frontend-normalized on read (string→array) in BOTH `use-profile.ts` translateInbound AND the prospect-profile adapter in `profile/[uuid]/page.tsx`.

**Compatibility.** `computeCompatibility(viewer, candidate)` (`src/lib/scoring/compute-compatibility.ts`) → `{score:0..100, breakdown}`. `CompatPill` renders the % (variant: ≥85 lime, ≥65 lavender, else pink); pass only `score` for a plain pill, add `breakdown` for the tap-to-expand sheet.

**Map.** react-leaflet (dynamic import, ssr:false). Markers = `MapAvatar` (`L.divIcon`, raw HTML string — escape user strings via `escapeAttr`). Marker click → `/profile/[id]?from=map`. Desktop hover popup = `MapProfilePopup` inside react-leaflet `<Popup className="ahavah-map-popup">` (Leaflet popup chrome neutralized in `globals.css`).
